## How to Use These Components

To use these components in your own MDX posts:

1. Save your post as `.mdx` instead of `.md` in the `_posts` directory
2. Import components using the global scope (automatically available)
3. Use the component syntax: `<ComponentName prop1={value} />`

### Example

```mdx
---
title: "My Interactive Post"
slug: "/2025/01/my-post"
date: "2025-01-15"
category: "Guide"
tags: ["mdx", "interactive"]
---

# My Interactive Post

Here's a tool comparison:

<ToolComparison
  tools={[
    {
      name: "Tool A",
      description: "A great tool",
      pros: ["Fast", "Easy"],
      cons: ["Expensive"],
      rating: 4.5
    }
  ]}
/>

And that's it!
```

## Summary

These 7 components provide powerful ways to create interactive, data-driven content:

- **ToolComparison**: Compare tools with ratings and pros/cons
- **FeatureMatrix**: Sortable tables with visual scoring
- **WakaTimeChart**: Stacked area charts for time data
- **ToolTimeline**: Sequential event timelines
- **WorkflowDiagram**: Process flows with arrows
- **VersionDiff**: Code difference viewer
- **ToolList**: Searchable, filterable tool databases

All components are responsive, accessible, and work seamlessly with the existing blog infrastructure. Enjoy creating rich, interactive content with MDX!
