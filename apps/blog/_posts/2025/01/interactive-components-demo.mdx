Welcome to the **Interactive Components Demo**! This post showcases all the new MDX components available for creating rich, interactive content in your blog.

The blog now supports both traditional `.md` files and modern `.mdx` files with React components. This flexibility allows you to choose the right approach for your content.

## What is MDX?

MDX is a format that lets you write JSX inside your Markdown content. It allows you to import and use React components directly in your markdown, making your content interactive and dynamic.

## Usage Guide

### Adding MDX to Your Content

To use these components in your own posts:

1. **Create a `.mdx` file** in the `_posts` directory with proper frontmatter
2. **Import components** (they're automatically available)
3. **Use components** in your MDX content with JSX syntax

### File Structure

```
_posts/
  2025/
    01/
      my-post.mdx
```

### Frontmatter Template

```mdx
---
slug: /2025/01/my-post
title: "My Amazing Post"
date: 2025-01-15
category: Development
tags: [react, components, mdx]
description: "Post description for SEO"
---

Your content here...
```

### Component Props

Each component has specific props for customization. Check the component source files for detailed prop interfaces and usage examples.

## Why MDX?

MDX provides several advantages:

- **Interactive Content**: Embed React components directly
- **Type Safety**: Full TypeScript support
- **Reusability**: Create custom components for your specific needs
- **Performance**: Client-side hydration only when needed
- **Flexibility**: Mix static content with dynamic elements

## Backward Compatibility

All existing `.md` posts continue to work exactly as before. The MDX support is additive - you can choose the format that works best for each post.
