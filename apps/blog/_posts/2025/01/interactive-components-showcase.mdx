## 6. Version Diff Viewer

Compare code versions with a Git-style diff viewer.

### Single File Diff

<VersionDiff
  title="API Endpoint Update"
  before={`// Old endpoint
app.get('/api/users/:id', async (req, res) => {
  const user = await Users.findById(req.params.id);
  if (!user) return res.status(404).json({ error: 'Not found' });
  return res.json(user);
});`}
  after={`// Updated endpoint with validation
app.get('/api/users/:id', async (req, res) => {
  try {
    const user = await Users.findById(req.params.id);
    if (!user) return res.status(404).json({ error: 'User not found' });
    return res.json({ data: user, timestamp: new Date() });
  } catch (error) {
    return res.status(500).json({ error: 'Server error' });
  }
});`}
  language="javascript"
/>

## Using Components in Your Posts

To use these components in your MDX posts:

1. **Import the component** (automatically available in MDX context)
2. **Pass props** as needed
3. **Customize** the content and data

### Example Usage

```mdx
<ToolComparison
  tools={[
    {
      name: "My Tool",
      pros: ["Fast", "Reliable"],
      cons: ["Complex setup"],
      rating: 4.5
    }
  ]}
/>
```

## Benefits of Interactive Content

- **Better Engagement**: Readers interact with dynamic content
- **Clearer Comparisons**: Visual aids make complex decisions easier
- **Memorable Content**: Interactive elements stick in memory
- **Rich Documentation**: Perfect for technical tutorials and reviews

## Conclusion

These components open up new possibilities for creating engaging technical content. They work seamlessly alongside traditional Markdown, so you can mix and match based on your content needs.

All existing `.md` posts continue to work without modification, ensuring backward compatibility.
