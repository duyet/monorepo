# MDX Components Showcase

Welcome to this comprehensive guide showcasing all 7 interactive MDX components available for use in our blog posts. These components are designed to enhance content creation by providing rich, interactive elements that engage readers.

## Overview

Our MDX support allows you to use React components directly in your markdown files, enabling:

- Interactive data visualizations
- Component-based UIs within content
- Rich comparisons and analyses
- Real-time user interaction

Let's explore each component with practical examples.

---

## 6. VersionDiff

Display code changes in Git-style diff format.

```jsx
<VersionDiff
  fileName="main.js"
  oldVersion={`const greeting = "Hello";
console.log(greeting);`}
  newVersion={`const greeting = "Hello, MDX!";
const name = "World";
console.log(greeting + " " + name);`}
/>
```

<VersionDiff
  fileName="utils.ts"
  oldVersion={`export const add = (a: number, b: number) => {
  return a + b;
}

export const subtract = (a: number, b: number) => {
  return a - b;
}`}
  newVersion={`export const add = (a: number, b: number): number => {
  return a + b;
}

export const subtract = (a: number, b: number): number => {
  return a - b;
}

export const multiply = (a: number, b: number): number => {
  return a * b;
}

export const divide = (a: number, b: number): number => {
  if (b === 0) throw new Error("Division by zero");
  return a / b;
}`}
/>

## Benefits of MDX Components

Using these interactive components in your MDX posts provides:

- **Enhanced Engagement**: Readers can interact with content instead of passive reading
- **Better Data Visualization**: Complex information becomes easier to understand
- **Rich Comparisons**: Make informed decisions with interactive comparison tables
- **Progress Tracking**: Visualize workflows and completion status
- **Code Understanding**: Diff viewers make changes clear and readable

## Getting Started

To use these components in your MDX posts:

1. **MDX Files**: Save your posts with `.mdx` extension in the `_posts` directory
2. **Import Components**: Components are auto-imported (or explicitly import from `@duyet/components`)
3. **Use in Content**: Just use JSX syntax directly in your markdown
4. **Front Matter**: Use standard front matter for metadata

## Conclusion

MDX transforms static content into interactive experiences. Whether you're writing tutorials, documenting processes, or sharing data insights, these components help you create more engaging and effective content.

Try each component above to see how they enhance the reading experience!
