## Writing Your Own MDX

To use these components in your own MDX posts:

1. Create a file with `.mdx` extension in `_posts/YYYY/MM/`
2. Add the frontmatter with `slug`, `title`, `date`, etc.
3. Use any of the 7 components directly in your content

Example:
\`\`\`mdx
---
slug: /2025/01/my-post
title: My MDX Post
---

# Hello MDX

<ToolComparison tools={[...]} />
\`\`\`

## Conclusion

MDX support brings powerful interactivity to our blog while maintaining backward compatibility with existing `.md` files. You can now create rich, interactive content that enhances the reader experience.

The 7 interactive components provide:
- **ToolComparison**: Side-by-side tool comparisons
- **FeatureMatrix**: Sortable comparison tables
- **WakaTimeChart**: Activity visualization
- **ToolTimeline**: Progress tracking
- **WorkflowDiagram**: Process illustration
- **VersionDiff**: Code comparison
- **ToolList**: Filterable tool directories

Happy writing!
