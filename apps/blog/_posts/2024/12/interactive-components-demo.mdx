# MDX Interactive Components Demo

Welcome to the new MDX-powered blog! This post demonstrates all 7 interactive components that you can now use in your .mdx posts. These components bring rich, interactive content to your blog while maintaining compatibility with existing .md posts.

## Tool Comparison

The **ToolComparison** component helps you compare two tools side-by-side with pros, cons, and ratings:

  tool1="React"
  tool2="Vue"

## Version Diff

Show code changes with the **VersionDiff** component:

<VersionDiff
  title="API Changes"
  filename="src/api/users.ts"
  lines={[
    { type: "header", content: "diff --git a/src/api/users.ts b/src/api/users.ts" },
    { type: "header", content: "--- a/src/api/users.ts" },
    { type: "header", content: "+++ b/src/api/users.ts" },
    { oldLine: 1, newLine: 1, content: "interface User {", type: "context" },
    { oldLine: 2, newLine: 2, content: "  id: string;", type: "context" },
    { oldLine: 3, newLine: 3, content: "  name: string;", type: "context" },
    { oldLine: 4, newLine: 4, content: "  email: string;", type: "context" },
    { oldLine: 5, newLine: 5, content: "}", type: "context" },
    { oldLine: 6, newLine: 6, content: " ", type: "context" },
    { oldLine: 7, newLine: 7, content: "-const getUsers = () => {", type: "removed" },
    { oldLine: 8, newLine: 8, content: "-  return fetch('/api/users').then(res => res.json());", type: "removed" },
    { oldLine: 9, newLine: 9, content: "};", type: "context" },
    { oldLine: 10, newLine: 10, content: " ", type: "context" },
    { oldLine: 11, newLine: 11, content: "+const getUsers = async (): Promise<User[]> => {", type: "added" },
    { oldLine: 12, newLine: 12, content: "+  const response = await fetch('/api/users');", type: "added" },
    { oldLine: 13, newLine: 13, content: "+  if (!response.ok) throw new Error('Failed to fetch');", type: "added" },
    { oldLine: 14, newLine: 14, content: "+  return response.json();", type: "added" },
    { oldLine: 15, newLine: 15, content: "+};", type: "added" }
  ]}

## Using Custom MDX Components

You can also import additional components directly in your MDX files:

```jsx
import { BarChart, Bar } from 'recharts'

<BarChart data={[{ name: 'A', value: 400 }, { name: 'B', value: 300 }]}>
  <Bar dataKey="value" fill="#3b82f6" />
</BarChart>
```

## Backward Compatibility

All existing `.md` posts continue to work without any changes. The system automatically:

1. Detects file type (`.md` vs `.mdx`)
2. Processes markdown normally for `.md` files
3. Renders MDX components for `.mdx` files
4. Falls back gracefully if MDX isn't configured
